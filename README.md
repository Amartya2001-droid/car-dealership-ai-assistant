# Car Dealership AI Assistant - Monitoring Dashboard Frontend

> **Frontend-only implementation** for the Car Dealership AI Assistant monitoring dashboard. This React application provides a premium operations interface for dealership staff to monitor leads, appointments, and follow-ups generated by the after-hours AI call assistant.

## 🎯 Project Overview

This repository contains the **frontend monitoring dashboard** that connects to the existing **Node.js Express backend** from the [car-dealership-ai-assistant](https://github.com/Amartya2001-droid/car-dealership-ai-assistant) repository.

**Important:** This is a **frontend-only** project. The backend endpoints must be running separately from the GitHub repository.

## 🚀 Quick Start

### Prerequisites
1. **Node.js 18+** and **Yarn** installed
2. **Express backend** running (from the GitHub repository)
3. Backend accessible at a known URL

### Installation

```bash
# Navigate to frontend directory
cd /app/frontend

# Install dependencies
yarn install

# Start development server
yarn start
```

The dashboard will be available at `http://localhost:3000`.

### Configuration

Edit `/app/frontend/.env` to set your backend URL:

```env
REACT_APP_BACKEND_URL=http://localhost:3000
# or your deployed backend URL
# REACT_APP_BACKEND_URL=https://your-backend.com
```

**⚠️ Important:** Never hardcode URLs in the code. Always use `process.env.REACT_APP_BACKEND_URL`.

## 🔌 Backend API Integration

The frontend consumes these Express backend endpoints:

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/health` | GET | Service health check |
| `/admin/runtime` | GET | Runtime configuration |
| `/admin/summary` | GET | Aggregated KPI data |
| `/admin/leads` | GET | All lead records |
| `/admin/appointments` | GET | Test drive appointments |
| `/admin/followups` | GET | Follow-up queue |

**Note:** All endpoints must be prefixed with the backend base URL from `REACT_APP_BACKEND_URL`.

## 🐛 Troubleshooting

### Dashboard shows "Unable to Load Dashboard"
**Cause:** Express backend not running or not accessible

**Solutions:**
1. Start the Express backend from the GitHub repository
2. Verify backend URL in `.env` is correct
3. Check CORS settings on backend allow frontend origin
4. Test backend endpoints directly: `curl http://localhost:3000/health`

### Empty data / No leads showing
**Cause:** Backend has no data yet

**Solutions:**
1. Use `/simulate/call` endpoint to add test leads (see TESTING_GUIDE.md)
2. Verify backend data files aren't empty
3. Wait for auto-refresh (30s) or click manual refresh

### Styling broken / Looks wrong
**Cause:** CSS not loading or browser cache

**Solutions:**
1. Clear browser cache and hard reload (Cmd+Shift+R / Ctrl+Shift+R)
2. Check browser console for CSS loading errors
3. Verify Tailwind CSS is building correctly
4. Restart frontend: `sudo supervisorctl restart frontend`

## 🤝 Integration Notes

### Backend Requirements
The Express backend must:
- ✅ Serve the 6 required API endpoints
- ✅ Return JSON in the expected format
- ✅ Allow CORS from the frontend origin
- ✅ Run on the port/URL specified in `REACT_APP_BACKEND_URL`

### Frontend Assumptions
The dashboard assumes:
- ✅ Backend endpoints exist at base URL + path (e.g., `${BACKEND_URL}/admin/leads`)
- ✅ Lead objects have standard fields (id, phone, callerName, inquiry, persona, mood, topic, urgency, etc.)
- ✅ Timestamps are in ISO 8601 format
- ✅ Arrays may be empty (graceful empty states)

## 🚢 Deployment

### Environment Variables
Production deployment requires:
```env
REACT_APP_BACKEND_URL=https://your-production-backend.com
WDS_SOCKET_PORT=443
ENABLE_HEALTH_CHECK=false
```

### Build Process
```bash
# Create production build
yarn build

# Output will be in /app/frontend/build/
# Serve with any static hosting (Vercel, Netlify, S3, etc.)
```

### CORS Configuration
Ensure Express backend allows your frontend origin:
```javascript
// In Express backend
app.use(cors({
  origin: 'https://your-frontend-domain.com',
  credentials: true
}));
```
